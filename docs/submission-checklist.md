# Hackathon Submission Checklist

## Core submission assets

- Project title
- Team member names
- Short description
- Detailed explanation
- Live demo URL
- GitHub repository URL
- 2-3 minute demo video
- API usage description

## Suggested title

Quran Journeys for Busy Professionals

## Suggested one-liner

A personalized AI-assisted Quran companion that helps busy professionals read, understand, and reflect in short daily sessions.

## Demo script outline (2-3 minutes)

1. Show onboarding and goal setup.
2. Generate 7-day plan and explain adaptive chunk sizing.
3. Walk through Read (Arabic + translation + audio).
4. Run Understand and show key points + simple summary.
5. Capture Reflect note + mood tag + feedback.
6. Complete session and show updated dashboard metrics.

## API usage summary

Content API:
- Uthmani verses endpoint
- Translation endpoint

User-oriented features currently implemented in app:
- Streak tracking
- Progress tracking
- Session logs and adaptation

## Judging optimization checklist

- Impact: clearly explain post-Ramadan consistency problem and how 10-20 minute sessions solve it.
- Product quality: ensure mobile demo is smooth and readable.
- Technical execution: run lint and build before recording demo.
- Innovation: emphasize adaptive planning and professional persona-aware explanation.
- API depth: mention live Quran Foundation content fetch and fallback resilience.
